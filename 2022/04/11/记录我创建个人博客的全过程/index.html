记录我创建个人博客的全过程

2022年4月11日 我创建了自己的第一个个人博客。虽然还有很多方面没有完善，但是我也是想记录一下自己在开荒摸索的全过程，以供个人记录和遵循互联网的开源精神。

软件需求

dev-sidecar：我用过体验最好的上github的软件，也可以加速npm。
下载方式：dev-sidecar: 开发者边车，github打不开，github 加速, git clone加速，google CDN加速，Stack Overflow加速 (gitee.com)

hexo：hexo是我用来构建博客的框架。
官网在这：Hexo，暂时不需要下载。

git：windows的终端软件，虽然也可以用自带终端，但是很多教程都用的git。
官网在这：Git (git-scm.com)

node.js：构建博客需要的运行环境。
我虽然不是在这下的，但是在这下肯定没问题：nodejs/node: Node.js JavaScript runtime (github.com)

可选visual studio code。

将网站部署到GitHub

这里我先不叙述如何美化博客，先叙述如何将个人博客部署到GitHub上。

1.在任何页面的右上角，使用 下拉菜单选择 New repository（新建仓库）

2.输入 username.github.io 作为存储库名称。 将 username 替换为您的 GitHub 用户名。 例如，如果您的用户名是 octocat，则存储库名称应为 octocat.github.io。仓库要设置为公开。

5.生成本地的SSH公钥并且绑定到远端的github，实现免密码登录（这一步很重要否则每次提交都需要输入密码）

将库同步到vercel

Vercel是一个静态网页部署网站，并不托管代码。因此，我们通过GitHub来托管 hexo deploy 生成的网页文件，并设置自动导入。当你的源代码仓库有变动后，Vercel会第一时间检测到并重新部署网站，部署完成后会通过邮件提醒。

进入Vercel官网，点击 Continue with GitHub

授权完成后，选择 Import Git Repository，点击Continue，进入如下界面。输入你的仓库 url（https）。
